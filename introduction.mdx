## What is alexa-apl-interface?

A complete TypeScript implementation of the Alexa Presentation Language (APL) Interface Reference. This library provides type-safe directive builders, request handlers, context parsers, validators, and a request router for building rich Alexa skill experiences.

## Supported Interfaces

| Interface | Directives | Requests | Description |
|---|---|---|---|
| `Alexa.Presentation.APL` | RenderDocument, ExecuteCommands, SendIndexListData, SendTokenListData, UpdateIndexListData | UserEvent, LoadIndexListData, LoadTokenListData, RuntimeError | Screen devices (Echo Show, Fire TV) |
| `Alexa.Presentation.APLA` | RenderDocument | RuntimeError | Audio rendering |
| `Alexa.Presentation.APLT` | RenderDocument, ExecuteCommands | — | Character displays (Echo Dot clock) |
| `Alexa.Presentation.HTML` | Start, HandleMessage | Message | Web content on screen devices |
| `AudioPlayer` | Play, Stop, ClearQueue | PlaybackStarted, PlaybackFinished, PlaybackStopped, PlaybackNearlyFinished, PlaybackFailed | Long-form audio streaming |
| `VideoApp` | Launch | PlaybackStarted, PlaybackFinished, PlaybackStopped, PlaybackFailed | Full-screen native video |
| `Alexa.DataStore` | — | DataStoreError | Data store error notifications |
| `Alexa.DataStore.PackageManager` | — | UsagesInstalled, UsagesRemoved, UpdateRequest, InstallationError | Widget package lifecycle |

## Key Features

- **Type-safe builders** — Fluent builder pattern with compile-time validation for all directives
- **Request type guards** — Narrow request types safely with `isAPLUserEvent()`, `isAudioPlayerPlaybackFailed()`, etc.
- **Context parsers** — Extract viewport capabilities, visual context hierarchies, and widget state
- **Validation utilities** — Runtime validation for directives and request timestamp freshness
- **Request router** — Route incoming requests to the correct handler with a single configuration object
- **Zero dependencies** — Only requires TypeScript ≥ 5.3

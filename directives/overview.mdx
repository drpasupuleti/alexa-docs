Directives are instructions sent from your skill to the Alexa device. This library provides two patterns for building directives:

1. **Builder classes** — Fluent API with method chaining for complex directives
2. **Factory functions** — One-liner convenience functions for common cases

Both patterns validate required fields at build time and throw descriptive errors for missing or invalid properties.

## Builder vs Factory Pattern

<Tabs>
  <Tab title="Builder Pattern">
    Use builders when you need fine-grained control or are constructing directives conditionally:

    ```typescript
    const directive = new RenderDocumentDirectiveBuilder()
      .setToken("my-token")
      .setDocument(myDocument)
      .addDataSource("myData", myDataSource)
      .build();
    ```
  </Tab>
  <Tab title="Factory Function">
    Use factory functions for straightforward, one-shot directive creation:

    ```typescript
    const directive = createRenderDocumentDirective(
      "my-token",
      myDocument,
      { myData: myDataSource }
    );
    ```
  </Tab>
</Tabs>

## Validation

All builders and factory functions validate required fields before returning a directive. If a required field is missing, a descriptive `Error` is thrown:

```typescript
try {
  const directive = new RenderDocumentDirectiveBuilder().build();
} catch (error) {
  // Error: "APL RenderDocument directive requires a token"
}
```

You can also validate directives after construction using the [Directive Validator](/validators/directive-validator).
